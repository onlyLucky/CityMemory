# 城迹 - 软件需求规格说明

## 一、文档概述

### 1.1 文档目的
本文档详细描述"城迹"微信小程序的功能需求、业务规则、数据逻辑及非功能需求，为开发团队提供明确的实现依据。

### 1.2 产品概述
城迹是一款地理知识答题小程序，通过闯关游戏和随机挑战形式帮助用户学习祖国大好河山及全球国家城市知识，寓教于乐。

### 1.3 品牌理念
**极简风格，"迹"谐音"记"** —— 记录你探索世界的足迹，在答题中丈量祖国山河与全球城市。

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 星星 | 用户答题获得的评价单位，可半星，满分5星，完美6星 |
| 关卡 | 答题闯关的基本单位，每关包含若干题目 |
| 层级 | 用户闯关进度，层级越高解锁国家越多 |
| 区域主题 | 根据地理位置划分的主题（亚洲、中东、欧洲、非洲、美洲） |
| 门票 | 闯关或随机模式的入场次数 |
| 血量 | 随机模式的生命值，答错扣减 |
| 金币 | 随机模式专属货币，用于商店购买道具 |
| 正确率 | 单关卡答对题目数/总题目数 |

## 二、用户角色与场景

### 2.1 用户角色

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 普通用户 | 18岁以上成年人，对地理知识感兴趣 | 趣味学习、知识拓展、社交分享 |
| 访客 | 未登录用户 | 快速体验、了解产品 |

### 2.2 用户场景

#### 场景1：碎片化学习
> 用户在通勤途中打开小程序，进行10分钟答题闯关，学习地理知识，查看排行榜了解自己的排名。

#### 场景2：随机挑战
> 用户使用随机模式，选择题目类型或商店道具，挑战自己的地理知识极限，获取星星和金币奖励。

#### 场景3：社交分享
> 用户获得高星级评价后，将成绩分享到朋友圈，邀请好友一起答题。

#### 场景4：竞技排名
> 用户查看排行榜，发现好友排名靠前，激发竞争心理，继续答题提升排名。

## 三、功能需求详细说明

### 3.1 用户系统

#### 3.1.1 微信授权登录

**功能描述**：用户通过微信授权快速登录小程序

**入口**：首次打开小程序自动触发 / 个人中心点击登录

**操作流程**：
1. 用户打开小程序
2. 系统检测未登录状态
3. 显示授权登录弹窗
4. 用户点击"授权登录"
5. 获取微信用户基本信息（昵称、头像）
6. 登录成功，跳转至省份初始化页

**业务规则**：
- 授权登录为强制要求，未登录用户仅可查看启动页推荐
- 用户信息存储：openid、昵称、头像、注册时间
- 支持静默登录（已授权用户自动登录）

**数据字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | String | 用户唯一标识（系统生成） |
| openid | String | 微信openid |
| nickname | String | 用户昵称 |
| avatar | String | 用户头像URL |
| province | String | 用户选择省份 |
| total_stars | Decimal | 总星星数（支持半星） |
| level_count | INT | 已通关卡数 |
| created_at | DateTime | 注册时间 |
| last_login | DateTime | 最后登录时间 |

---

#### 3.1.2 省份初始化

**功能描述**：用户首次登录初始化省份，支持自动获取和手动选择

**入口**：首次登录后自动跳转

**操作流程**：

**方式一：自动获取**
1. 系统请求位置权限
2. 获取用户城市/省份模糊位置
3. 自动设置初始省份
4. 解锁该省份关卡
5. 跳转至首页

**方式二：手动选择**
1. 显示中国省份地图/列表
2. 用户选择居住省份
3. 系统保存选择
4. 解锁该省份关卡
5. 跳转至首页

**业务规则**：
- 优先推荐自动获取方式
- 用户可选择切换为手动选择
- 初始化后不可再次初始化（可在设置中修改省份）
- 选择省份后，自动解锁该省份关卡
- 推荐算法优先展示相关省份题目

**数据字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| user_province | String | 用户选择省份 |
| unlocked_provinces | Array | 已解锁省份列表 |
| unlocked_countries | Array | 已解锁国家列表 |
| init_method | String | 初始化方式：auto/manual |

#### 3.3.3 本地题目生成算法

**生成流程**：

```
1. 获取关卡配置
   - 区域主题（亚洲/中东/欧洲/非洲/美洲）
   - 难度等级
   - 题型比例
   
2. 获取国家/城市数据
   - 从本地缓存获取
   - 或从远程API动态获取
   
3. 生成题目
   - 随机选择目标城市/国家/古称
   - 生成干扰项（同区域/同级别）
   - 打乱选项顺序
   
4. 缓存已生成题目
   - 记录已出题目ID
   - 避免同一关卡重复
```

**干扰项生成规则**：

| 题型 | 干扰项来源 |
|------|------------|
| 城市猜省份 | 同区域省份、相邻省份 |
| 城市猜国家 | 同区域国家、知名度相近国家 |
| 古称猜今名 | 同时期古称对应城市 |
| 国旗猜国家 | 同区域国家、国旗相似国家 |

### 3.4 星级系统

#### 3.4.1 星级评价规则

**基本规则**：

| 属性 | 说明 |
|------|------|
| 评价单位 | 星星（可半星，如3.5星） |
| 满分 | 5星 |
| 完美评价 | 6星（100%正确率 + 时间达标） |
| 重复闯关 | 仅评星增加时才增加星级 |

**星级计算公式**：

```
基础星级计算：
- 正确率 100% → 5星
- 正确率 90-99% → 4.5星
- 正确率 80-89% → 4星
- 正确率 70-79% → 3.5星
- 正确率 60-69% → 3星
- 正确率 50-59% → 2.5星
- 正确率 40-49% → 2星
- 正确率 30-39% → 1.5星
- 正确率 20-29% → 1星
- 正确率 <20% → 0.5星

时间加成计算：
- 平均答题时间 ≤ 5秒 → +1星
- 平均答题时间 5-10秒 → +0.5星
- 平均答题时间 > 10秒 → +0星

最终星级 = min(基础星级 + 时间加成, 6星)
```

**星级示例**：

| 正确率 | 平均用时 | 基础星级 | 时间加成 | 最终星级 |
|--------|----------|----------|----------|----------|
| 100% | 4秒 | 5星 | +1星 | 6星 ⭐ |
| 100% | 8秒 | 5星 | +0.5星 | 5.5星 |
| 80% | 6秒 | 4星 | +0.5星 | 4.5星 |
| 60% | 12秒 | 3星 | +0星 | 3星 |

---

#### 3.4.2 星级更新规则

**重复闯关规则**：
- 用户可重复挑战已通关卡
- 仅当新星级 > 原星级时，更新星级
- 星级差值累加到总星星数

**示例**：
```
原星级：3星
新星级：4.5星
星级增加：1.5星
总星星数：+1.5星

原星级：4星
新星级：3.5星
星级不增加
总星星数：不变
```

### 3.6 答题系统

#### 3.6.1 闯关答题

**功能描述**：用户通过答题闯关，解锁新关卡

**入口**：首页点击"开始答题" / 关卡地图选择关卡

**操作流程**：
1. 用户选择关卡
2. 检查门票（门票不足则提示）
3. 扣除1张门票
4. 系统生成10道题目
5. 显示题目（无倒计时）
6. 用户选择答案
7. 即时反馈正确/错误
8. 完成所有题目后显示结果
9. 计算星级，判定闯关结果
10. 成功则解锁下一关

**业务规则**：
- 每关10道题目
- 无倒计时限制
- 答题过程中可退出（视为放弃，不返还门票）
- 同一关卡可重复挑战，取最高星级
- 门票不足时无法开始答题

---

#### 3.6.2 闯关判定

**判定规则**：

| 正确率 | 判定结果 | 后续操作 |
|--------|----------|----------|
| ≥60% | 闯关成功 | 解锁下一关，获得星级评价 |
| <60% | 闯关失败 | 可重新挑战（需门票） |

**失败重试规则**：
- 失败后可立即重试
- 重试生成新题目
- 需要消耗门票

### 3.7 随机模式系统

#### 3.7.1 随机模式概述

**功能描述**：无限答题模式，固定血量，挑战极限

**入口**：首页点击"随机模式"

**核心机制**：

| 属性 | 说明 |
|------|------|
| 血量 | 初始10点，答错-1 |
| 结束条件 | 血量为0 |
| 每日次数 | 3次 |
| 题目类型 | 4种题型（含国旗猜国家） |
| 题目来源 | 根据闯关层级提供相关国家题目 |
| 奖励 | 金币 + 星星 |

---

#### 3.7.2 随机模式流程

**进入流程**：
1. 用户点击"随机模式"
2. 检查门票（每日3次）
3. 扣除1张门票
4. 显示选项界面（3个随机选项）
5. 用户选择后开始答题

**选项随机生成机制**：

每次进入随机模式，系统随机生成3个选项，选项池包含：

| 选项池 | 出现概率 | 说明 |
|--------|----------|------|
| 题目类型-简单 | 25% | 10题，金币×1.0 |
| 题目类型-中等 | 20% | 15题，金币×1.5 |
| 题目类型-困难 | 15% | 20题，金币×2.0 |
| 商店 | 20% | 可购买血瓶、护盾、道具卡 |
| 血池 | 20% | 增加初始血量 |

**选项类型详解**：

| 选项类型 | 说明 | 效果 |
|----------|------|------|
| 题目类型-简单 | 基础难度 | 10题，金币×1.0，适合新手 |
| 题目类型-中等 | 进阶难度 | 15题，金币×1.5，适合有经验用户 |
| 题目类型-困难 | 高级难度 | 20题，金币×2.0，适合高手挑战 |
| 商店 | 道具购买 | 使用金币购买血瓶、护盾、道具卡 |
| 血池 | 额外血量 | 初始血量+3（变为13点） |

**血池效果说明**：
- 选择血池选项后，初始血量从10点增加至13点
- 血池效果仅当前随机模式有效
- 血池选项可与其他选项叠加（先选血池，再选题目类型）

**答题流程**：
1. 显示题目（无倒计时）
2. 用户选择答案
3. 正确 → 继续下一题
4. 错误 → 血量-1 → 显示正确答案
5. 血量为0 → 结束
6. 计算奖励（金币 + 星星）

---

#### 3.7.3 随机模式奖励

**金币奖励**：

```
金币 = 答对题目数 × 难度系数 × 10

难度系数：
- 简单：1.0
- 中等：1.5
- 困难：2.0
```

**星星奖励**：

```
星星 = 答对题目数 × 0.1
（向上取整到半星）
```

### 3.9 关卡系统

#### 3.9.1 层级结构

**层级设计**：

```
第1层：用户选择省份（初始解锁）
    ↓
第2层：相邻省份（闯关成功解锁）
    ↓
第3层：全国省份（逐步解锁）
    ↓
第4层：亚洲国家（解锁条件：完成20个关卡）
    ↓
第5层：中东国家（解锁条件：完成25个关卡）
    ↓
第6层：欧洲国家（解锁条件：完成30个关卡）
    ↓
第7层：非洲国家（解锁条件：完成35个关卡）
    ↓
第8层：美洲国家（解锁条件：完成40个关卡）
```

## 七、业务流程图

### 7.1 用户注册流程

```
开始
  ↓
打开小程序
  ↓
检测登录状态 → 已登录 → 检测初始化状态
  ↓ 未登录                ↓ 已初始化
显示授权弹窗              进入首页
  ↓                       ↓ 未初始化
用户授权 → 拒绝 → 提示需要授权  显示省份初始化页
  ↓ 同意                       ↓
获取用户信息            自动获取位置 → 成功 → 自动设置省份
  ↓                       ↓ 失败
检测初始化状态            手动选择省份
  ↓                       ↓
跳转首页                  解锁初始关卡
  ↓                       ↓
结束                      跳转首页
```

### 7.2 闯关答题流程

```
开始
  ↓
选择关卡
  ↓
检测门票 → 不足 → 提示购买/等待恢复
  ↓ 足够
扣除1张门票
  ↓
生成10道题目
  ↓
显示题目（无倒计时）
  ↓
用户选择答案
  ↓
判断正确性 → 正确 → 显示正确反馈
  ↓ 错误
显示正确答案
  ↓
记录答题数据
  ↓
是否最后一题 → 否 → 下一题
  ↓ 是
计算星级
  ↓
计算正确率
  ↓
正确率≥60%? → 是 → 闯关成功 → 解锁下一关 → 更新星级
  ↓ 否
闯关失败
  ↓
结束
```

### 7.3 随机模式流程

```
开始
  ↓
检测门票 → 不足 → 提示明日再来
  ↓ 足够
扣除1张门票
  ↓
显示选项界面（3个选项）
  ↓
用户选择选项
  ├─ 题目类型 → 设置难度
  ├─ 商店 → 购买道具
  └─ 血池 → 增加血量
  ↓
开始答题（血量10）
  ↓
显示题目（无倒计时）
  ↓
用户选择答案
  ↓
判断正确性 → 正确 → 继续答题
  ↓ 错误
血量-1 → 血量是否为0?
  ↓ 否                    ↓ 是
显示正确答案              结束答题
  ↓                       ↓
继续答题                  计算奖励（金币+星星）
  ↓                       ↓
（循环）                  结束
```
