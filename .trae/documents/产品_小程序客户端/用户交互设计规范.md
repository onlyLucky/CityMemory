# 城迹 - 用户交互设计规范

## 一、文档概述

### 1.1 文档目的
本文档定义"城迹"微信小程序的交互设计规范，包括页面结构、操作流程、交互逻辑及体验原则，为UI设计师和开发团队提供设计依据。

### 1.2 品牌理念
**极简风格，"迹"谐音"记"** —— 记录你探索世界的足迹，在答题中丈量祖国山河与全球城市。

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 极简设计 | 界面简洁，操作直观，减少认知负担 |
| 即时反馈 | 每个操作都有明确反馈 |
| 游戏化体验 | 星级评价、随机模式、商店系统增强趣味性 |
| 知识优先 | 内容呈现清晰，学习体验流畅 |

### 1.4 设计规范

#### 1.4.1 色彩规范

| 色彩类型 | 色值 | 应用场景 |
|----------|------|----------|
| 主色调 | #2E7D32 | 主按钮、导航栏、重要元素 |
| 辅助色 | #4CAF50 | 次要按钮、成功状态 |
| 强调色 | #FF9800 | 提示、警告、血量 |
| 背景色 | #F5F5F5 | 页面背景 |
| 文字主色 | #212121 | 主要文字 |
| 文字辅色 | #757575 | 次要文字、说明文字 |
| 成功色 | #4CAF50 | 正确答案、闯关成功 |
| 失败色 | #F44336 | 错误答案、闯关失败 |
| 星星色 | #FFD700 | 星级评价 |

#### 1.4.2 字体规范

| 字体类型 | 字号 | 字重 | 应用场景 |
|----------|------|------|----------|
| 大标题 | 24px | Bold | 页面标题 |
| 标题 | 20px | Medium | 模块标题 |
| 正文 | 16px | Regular | 正文内容 |
| 辅助文字 | 14px | Regular | 说明文字 |
| 小字 | 12px | Regular | 标签、时间 |

#### 1.4.3 间距规范

| 间距类型 | 数值 | 应用场景 |
|----------|------|----------|
| 页面边距 | 32rpx | 页面左右边距 |
| 模块间距 | 40rpx | 模块之间间距 |
| 元素间距 | 24rpx | 同一模块内元素间距 |
| 文字间距 | 16rpx | 文字与图标间距 |

## 二、页面结构

### 2.1 页面架构图

```
小程序
├── 启动页（splash）- 每日推荐
├── 授权页（auth）
├── 省份初始化页（province-init）
├── 首页（home）
│   ├── 闯关入口

│   └── 排行榜入口
├── 关卡地图页（level-map）

## 三、页面详细设计

| 元素 | 位置 | 说明 |

| 每日推荐卡片 | 中部 | 非首次进入显示 |
| 倒计时 | 右上角 | 倒计时数字 |
| 跳过按钮 | 右上角 | "跳过"按钮 |
| 查看详情按钮 | 卡片底部 | "查看详情"按钮 |

#### 页面布局（首次进入）

```
┌─────────────────────────┐
│                         │
│                         │
│         🏙️              │
│       城 迹             │
│                         │
│   记录你探索世界的足迹   │
│                         │
│                         │
│                         │
└─────────────────────────┘
```

#### 页面布局（非首次进入）

```
┌─────────────────────────┐
│                   跳过 5s│
├─────────────────────────┤
│                         │
│  ┌─────────────────┐    │
│  │   [城市图片]    │    │
│  └─────────────────┘    │
│                         │
│  今日推荐               │
│  苏州 🏮                │
│  江苏省 | 中国          │
│                         │
│  📍 东方威尼斯          │
│                         │
│  苏州，古称姑苏...      │
│                         │
│  [查看详情]             │
│                         │
└─────────────────────────┘
```

#### 交互逻辑

```
进入启动页
  ↓
检测是否首次进入
  ↓
首次 → 显示品牌Logo（3秒）→ 跳转授权页
  ↓
非首次 → 显示每日推荐（5秒倒计时）
  ↓

├─ 点击"跳过" → 跳转首页
├─ 点击"查看详情" → 跳转推荐详情页
└─ 倒计时结束 → 自动跳转首页
```

#### 交互细节

| 操作 | 反馈 |
|------|------|
| 点击跳过 | 立即跳转首页 |
| 点击查看详情 | 跳转推荐详情页 |
| 倒计时结束 | 自动跳转首页 |

### 3.8 随机模式入口页（random-entry）

#### 页面目的
随机模式选项选择

#### 页面元素

| 元素 | 位置 | 说明 |
|------|------|------|
| 返回按钮 | 左上角 | 返回首页 |
| 标题 | 顶部 | "随机模式" |
| 门票显示 | 顶部 | 今日剩余次数 |
| 选项卡片 | 中部 | 3个随机选项（从选项池随机生成） |
| 开始按钮 | 底部 | "开始挑战"按钮 |

#### 选项池说明

每次进入随机模式，系统从以下选项池随机生成3个选项：

| 选项池 | 出现概率 | 显示内容 |
|--------|----------|----------|
| 题目类型-简单 | 25% | 🎯 简单题目 \| 10题 \| 金币×1.0 |
| 题目类型-中等 | 20% | 🎯 中等题目 \| 15题 \| 金币×1.5 |
| 题目类型-困难 | 15% | 🎯 困难题目 \| 20题 \| 金币×2.0 |
| 商店 | 20% | 🏪 商店 \| 购买道具 |
| 血池 | 20% | ❤️ 血池 \| 初始血量+3 |

#### 页面布局（示例）

```
┌─────────────────────────┐
│ ← 随机模式   今日: 2/3  │
├─────────────────────────┤
│                         │
