# 城迹运营后台 - 软件需求规格说明

## 一、文档概述

### 1.1 文档目的
本文档详细描述"城迹"运营后台管理系统的功能需求、业务规则、数据逻辑及非功能需求，为开发团队提供明确的实现依据。

### 1.2 产品概述
城迹运营后台是为"城迹"微信小程序配套的管理系统，提供数据看板、用户管理、题库管理、内容管理、商店管理等核心功能，支撑产品运营决策。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| DAU | 日活跃用户数 |
| MAU | 月活跃用户数 |
| 留存率 | 用户在特定周期内再次访问的比例 |
| 管理员 | 后台系统的操作人员 |
| 角色 | 管理员权限分组 |
| 操作日志 | 管理员操作记录 |

## 三、功能需求详细说明

### 3.1 登录系统

#### 3.1.1 管理员登录

**功能描述**：管理员通过账号密码登录后台系统

**入口**：访问后台系统首页

**操作流程**：
1. 访问后台系统URL
2. 显示登录页面
3. 输入账号、密码
4. 点击"登录"
5. 验证成功后跳转数据看板

**业务规则**：
- 账号密码错误超过5次，锁定账号30分钟
- 登录成功后记录登录日志（IP、时间、设备）
- Session有效期2小时，超时需重新登录
- 支持记住登录状态（7天有效）

**数据字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| admin_id | String | 管理员唯一标识 |
| username | String | 登录账号 |
| password | String | 密码（加密存储） |
| role_id | String | 角色ID |
| status | Int | 状态：1启用/0禁用 |
| last_login_time | DateTime | 最后登录时间 |
| last_login_ip | String | 最后登录IP |

| 选项A | 文本输入 | 是 | 选项内容 |
| 选项B | 文本输入 | 是 | 选项内容 |

#### 3.4.3 题目审核

**功能描述**：审核新增或编辑的题目

**入口**：侧边栏"题库管理" → "题目审核"

**审核流程**：
1. 查看待审核题目列表
2. 点击题目查看详情
3. 审核通过/驳回
4. 驳回需填写原因

**审核规则**：
- 题目内容准确无误
- 选项无重复
- 正确答案正确
- 难度评估合理

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 3秒（首屏） |
| 列表查询响应 | < 1秒 |
| 数据导出 | < 30秒（10000条以内） |
| 并发用户数 | 支持10个管理员同时在线 |

### 4.2 安全需求

| 安全项 | 要求 |
|--------|------|
| 登录安全 | 密码加密存储、错误锁定、Session管理 |
| 权限控制 | 基于角色的权限控制（RBAC） |
| 操作审计 | 所有敏感操作记录日志 |
| 数据安全 | 敏感数据加密传输（HTTPS） |
| XSS防护 | 输入过滤、输出编码 |
| CSRF防护 | Token验证 |

### 4.3 可用性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | 99.5% |
| 故障恢复时间 | < 1小时 |
| 数据备份 | 每日自动备份 |

### 4.4 兼容性需求

| 平台 | 要求 |
|------|------|
| 浏览器 | Chrome 80+、Firefox 75+、Safari 13+、Edge 80+ |
