### 2.3 后端编码规范

#### 2.3.1 控制器规范

```typescript
import { Request, Response, NextFunction } from 'express'
import { UserService } from '@/services/admin/user.service'
import { success, error } from '@/utils/response'
import { logger } from '@/utils/logger'

export class UserController {
  private userService: UserService

  constructor() {
    this.userService = new UserService()
  }

  getList = async (req: Request, res: Response, next: NextFunction) => {
    try {
      const { page = 1, pageSize = 20, nickname, province } = req.query

      const result = await this.userService.getUserList({
        page: Number(page),
        pageSize: Number(pageSize),
        nickname: nickname as string,
        province: province as string
      })

      return success(res, result)
    } catch (err) {
      logger.error('获取用户列表失败', err)
      next(err)
    }
  }

  getDetail = async (req: Request, res: Response, next: NextFunction) => {
    try {
      const { id } = req.params

      const user = await this.userService.getUserById(id)

      if (!user) {
        return error(res, '用户不存在', 404)
      }

      return success(res, user)
    } catch (err) {
      logger.error('获取用户详情失败', err)
      next(err)
    }
  }
}

export default new UserController()
```

#### 2.3.2 服务层规范

```typescript
import { UserModel } from '@/models/mysql/user'
import { RedisService } from '@/services/common/cache.service'
import { logger } from '@/utils/logger'

interface UserListParams {
  page: number
  pageSize: number
  nickname?: string
  province?: string
}

interface UserListResult {
  list: User[]
  total: number
  page: number
  pageSize: number
}

export class UserService {
  private userModel: typeof UserModel
  private redisService: RedisService

  constructor() {
    this.userModel = UserModel
    this.redisService = new RedisService()
  }

  async getUserList(params: UserListParams): Promise<UserListResult> {
    const { page, pageSize, nickname, province } = params
    const offset = (page - 1) * pageSize

    const where: Record<string, unknown> = {}
    if (nickname) {
      where.nickname = { like: `%${nickname}%` }
    }
    if (province) {
      where.province = province
    }

    const [list, total] = await Promise.all([
      this.userModel.findAll({
        where,
        offset,
        limit: pageSize,
        order: [['created_at', 'DESC']]
      }),
      this.userModel.count({ where })
    ])

    return {
      list,
      total,
      page,
      pageSize
    }
  }

  async getUserById(id: string): Promise<User | null> {
    const cacheKey = `user:${id}`
    
    const cached = await this.redisService.get(cacheKey)
    if (cached) {
      return JSON.parse(cached)
    }

    const user = await this.userModel.findByPk(id)
    
    if (user) {
      await this.redisService.setex(cacheKey, 3600, JSON.stringify(user))
    }

    return user
  }
}

export default new UserService()
```

## 三、命名规范

### 3.1 文件命名

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| Vue组件 | PascalCase | `UserList.vue`, `QuestionCard.vue` |
| TypeScript文件 | camelCase | `useTable.ts`, `formatDate.ts` |
| 类型定义文件 | camelCase + .d.ts | `user.d.ts`, `api.d.ts` |
| 样式文件 | kebab-case | `variables.scss`, `reset.scss` |
| 图片资源 | kebab-case | `logo-primary.png`, `icon-user.svg` |
| 目录名 | kebab-case | `user-list/`, `question-edit/` |

### 3.2 变量命名

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 普通变量 | camelCase | `userName`, `totalCount` |
| 常量 | UPPER_SNAKE_CASE | `MAX_COUNT`, `API_BASE_URL` |
| 私有变量 | _camelCase | `_privateData`, `_internalState` |
| 布尔值 | is/has/can + camelCase | `isLoading`, `hasPermission`, `canEdit` |
| 数组 | 复数形式或List后缀 | `users`, `questionList` |
| 对象 | 单数形式 | `user`, `question` |
| 函数 | 动词开头 | `getUserList`, `handleSubmit`, `formatDate` |
| 异步函数 | async前缀或动词 | `asyncFetchData`, `loadUserData` |
| 事件处理 | handle + 事件名 | `handleClick`, `handleSubmit` |
| 回调函数 | on + 事件名 | `onSuccess`, `onError` |

### 3.3 组件命名

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 页面组件 | 功能名 + Page/View | `UserListPage`, `DashboardView` |
| 业务组件 | 功能描述 | `QuestionCard`, `UserDetail` |
| 通用组件 | 功能描述 | `Pagination`, `SearchForm` |
| 布局组件 | Layout后缀 | `DefaultLayout`, `BlankLayout` |
| 高阶组件 | with前缀 | `withAuth`, `withLoading` |

### 3.4 数据库命名

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 表名 | snake_case（复数） | `admins`, `roles`, `admin_logs` |
| 字段名 | snake_case | `user_name`, `created_at` |
| 主键 | id | `id` |
| 外键 | 表名_id | `admin_id`, `role_id` |
| 索引 | idx_表名_字段 | `idx_admins_status` |
| 唯一索引 | uk_表名_字段 | `uk_admins_username` |

### 3.5 API命名

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 获取列表 | GET /resources | `GET /api/v1/admin/users` |

| 创建 | POST /resources | `POST /api/v1/admin/questions` |

### 5.2 分支命名规范

| 分支类型 | 命名规范 | 示例 |
|----------|----------|------|
| 主分支 | main | main |
| 开发分支 | develop | develop |
| 功能分支 | feature/功能名 | feature/user-management |
| 发布分支 | release/版本号 | release/v1.0.0 |
| 热修复分支 | hotfix/修复描述 | hotfix/fix-login-bug |
| Bug修复分支 | fix/修复描述 | fix/user-list-pagination |

### 5.3 提交信息规范

**提交信息格式**
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type类型**
| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug修复 |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构（不是新功能也不是Bug修复） |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具相关 |
| revert | 回滚提交 |

**Scope范围**
| 范围 | 说明 |
|------|------|
| auth | 认证模块 |
| user | 用户管理 |
| question | 题库管理 |
| dashboard | 数据看板 |
| content | 内容管理 |
| shop | 商店管理 |
| system | 系统管理 |

**提交示例**
```
feat(user): 新增用户列表导出功能

- 支持按筛选条件导出
- 支持Excel格式导出
- 导出数量限制10000条

Closes #123
```

```
fix(question): 修复批量导入题目状态错误

批量导入的题目状态应该为"待审核"，之前错误设置为"启用"

Fixes #456
```

### 5.4 版本号规范

**语义化版本**
```
MAJOR.MINOR.PATCH

MAJOR: 不兼容的API变更
MINOR: 向后兼容的功能新增
PATCH: 向后兼容的问题修复
```

**版本示例**
| 版本 | 说明 |
|------|------|
| 1.0.0 | 首次正式发布 |
| 1.1.0 | 新增批量导入功能 |
| 1.1.1 | 修复批量导入Bug |

```

<type>(<scope>): <subject>

## 变更类型

- [ ] 无ESLint错误
- [ ] 无TypeScript类型错误
```

### 6.3 代码审查清单

**通用检查项**
| 检查项 | 说明 |
|--------|------|
| 代码规范 | 是否符合编码规范 |
| 命名规范 | 变量、函数、类命名是否合理 |
| 注释 | 是否有必要的注释 |
| 类型定义 | TypeScript类型是否完整 |
| 错误处理 | 是否正确处理错误 |
| 安全性 | 是否存在安全风险 |
| 性能 | 是否存在性能问题 |

**前端检查项**
| 检查项 | 说明 |
|--------|------|
| 组件拆分 | 组件是否合理拆分 |
| 状态管理 | 状态是否合理管理 |
| Props定义 | Props类型是否完整 |
| 事件处理 | 事件命名是否规范 |
| 样式规范 | 是否使用BEM命名 |
| 响应式 | 是否考虑响应式布局 |

**后端检查项**
| 检查项 | 说明 |
|--------|------|
| API设计 | 是否符合RESTful规范 |
| 参数校验 | 是否完整校验参数 |
| 数据库操作 | 是否有SQL注入风险 |
| 缓存策略 | 缓存是否合理使用 |
| 日志记录 | 是否记录必要日志 |
| 事务处理 | 事务是否正确处理 |

### 6.4 审查流程

1. **提交PR**
   - 确保代码已通过本地测试
   - 确保代码符合编码规范
   - 填写完整的PR描述

2. **自动检查**
   - ESLint检查
   - TypeScript类型检查
   - 单元测试运行
   - 构建检查

3. **人工审查**
   - 至少1位审查人员通过
   - 审查人员提出修改意见
   - 开发人员修改代码

4. **合并代码**
   - 所有检查通过
   - 审查人员批准
   - 合并到目标分支
   - 删除功能分支
