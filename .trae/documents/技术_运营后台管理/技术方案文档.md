## 二、技术选型

### 2.1 整体技术栈

| 层级 | 技术选型 | 版本要求 | 选型理由 |
|------|----------|----------|----------|
| 前端框架 | Vue 3 | 3.4+ | 组合式API，性能优异，生态成熟 |
| UI组件库 | Element Plus | 2.5+ | 企业级组件库，功能完善 |
| 状态管理 | Pinia | 2.1+ | Vue 3官方推荐，轻量级 |
| 构建工具 | Vite | 5.0+ | 快速热更新，开发体验好 |
| 类型系统 | TypeScript | 5.0+ | 类型安全，代码可维护性高 |
| 后端框架 | Node.js + Koa | 18.x LTS / 2.x | 与小程序后端统一技术栈 |
| 关系数据库 | MySQL | 8.0+ | 复用小程序数据库，数据一致性 |
| 缓存数据库 | Redis | 7.0+ | 会话管理、权限缓存 |
| 文档数据库 | MongoDB | 6.0+ | 题库、城市、国家数据存储 |
| 图表库 | ECharts | 5.4+ | 功能强大，数据可视化首选 |
| 对象存储 | 腾讯云COS | - | 微信生态友好，CDN加速 |
| 容器化 | Docker | 24.x | 标准化部署，便于扩展 |

### 2.2 前端技术选型详解

#### 2.2.1 为什么选择Vue 3 + Element Plus

| 对比项 | Vue 3 + Element Plus | React + Ant Design | Angular + NG-ZORRO |
|--------|---------------------|-------------------|-------------------|
| 学习成本 | 低 | 中 | 高 |
| 开发效率 | 高 | 高 | 中 |
| 生态成熟度 | 高 | 高 | 中 |
| 性能 | 优秀 | 优秀 | 良好 |
| TypeScript支持 | 优秀 | 优秀 | 优秀 |
| 组件丰富度 | 高 | 高 | 中 |

**结论**：Vue 3组合式API配合Element Plus组件库，开发效率高，适合后台管理系统快速开发。

## 七、安全设计

### 7.1 认证安全

| 安全措施 | 说明 |
|----------|------|
| 密码加密 | bcrypt加密存储，cost=10 |
| JWT认证 | RS256签名，Token有效期2小时 |
| 登录限制 | 错误5次锁定30分钟 |
| Token刷新 | Refresh Token机制，7天有效 |
| 会话管理 | Redis存储，支持主动失效 |

### 7.2 权限控制

| 控制层级 | 说明 |
|----------|------|
| 路由权限 | 前端路由守卫，无权限跳转403 |
| 接口权限 | 后端中间件验证，无权限返回403 |
| 数据权限 | 基于角色的数据访问控制 |
| 操作权限 | 敏感操作二次确认 |

### 7.3 数据安全

| 安全措施 | 说明 |
|----------|------|
| HTTPS传输 | 全站HTTPS加密 |
| SQL注入防护 | 参数化查询，ORM框架 |
| XSS防护 | 输入过滤，输出编码 |
| CSRF防护 | Token验证 |
| 敏感数据脱敏 | 日志脱敏，接口脱敏 |

### 7.4 操作审计

| 审计内容 | 说明 |
|----------|------|
| 登录日志 | 登录时间、IP、设备 |
| 操作日志 | 操作类型、模块、内容、结果 |
| 数据变更 | 敏感数据变更前后记录 |

## 八、性能优化方案
