# 城迹系统架构设计文档

## 一、文档概述

### 1.1 文档目的
本文档定义"城迹"项目的系统架构设计，包括总体架构、技术架构、部署架构、数据架构、安全架构、性能架构和扩展性架构，为项目开发和运维提供指导。

### 1.2 项目背景
"城迹"是一款地理知识问答类微信小程序，用户通过答题闯关的方式学习地理知识。项目包含小程序端、运营后台管理系统和统一的服务端。

### 1.3 架构目标

| 目标 | 说明 |
|------|------|
| 高可用性 | 系统可用性达到99.9%以上 |
| 高性能 | 响应时间小于200ms，支持10万+并发用户 |
| 可扩展性 | 支持水平扩展，轻松应对业务增长 |
| 安全性 | 保护用户数据安全，防止恶意攻击 |
| 可维护性 | 代码结构清晰，便于维护和升级 |

### 4.3 部署方案

#### 4.3.1 容器化部署

使用Docker容器化部署，提高部署效率和一致性。

```dockerfile
# 服务端Dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

EXPOSE 3000

CMD ["npm", "start"]
```

```yaml
# docker-compose.yml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DB_HOST=mysql
      - REDIS_HOST=redis
      - MONGODB_HOST=mongodb
    depends_on:
      - mysql
      - redis
      - mongodb

  mysql:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=root
      - MYSQL_DATABASE=citytrace
    volumes:
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:7.0
    volumes:
      - redis_data:/data

  mongodb:
    image: mongo:6.0
    volumes:
      - mongodb_data:/data/db

volumes:
  mysql_data:
  redis_data:
  mongodb_data:
```

#### 4.3.2 Kubernetes部署

使用Kubernetes进行容器编排，实现自动化部署和扩展。

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: citytrace-server
spec:
  replicas: 3
  selector:
    matchLabels:
      app: citytrace-server
  template:
    metadata:
      labels:
        app: citytrace-server
    spec:
      containers:
      - name: server
        image: citytrace-server:latest
        ports:
        - containerPort: 3000
        env:
        - name: NODE_ENV
          value: "production"
        - name: DB_HOST
          valueFrom:
            configMapKeyRef:
              name: citytrace-config
              key: db-host
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
```

### 4.4 负载均衡

使用Nginx作为负载均衡器，实现请求分发和故障转移。

```nginx
# nginx.conf
upstream citytrace_backend {
    least_conn;
    server server1:3000 weight=3;
    server server2:3000 weight=2;
    server server3:3000 weight=1;
    keepalive 32;
}

server {
    listen 80;
    server_name api.citytrace.com;

    location / {
        proxy_pass http://citytrace_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 6.2 身份认证

#### 6.2.1 小程序端认证

使用微信登录 + JWT令牌认证。

```
1. 小程序调用wx.login()获取code
2. 前端将code发送到服务端
3. 服务端使用code换取openid
4. 服务端查询或创建用户
5. 服务端生成JWT令牌
6. 前端存储令牌，后续请求携带令牌
```

#### 6.2.2 后台管理端认证

使用用户名密码 + JWT令牌认证。

```
1. 用户输入用户名密码
2. 前端发送到服务端
3. 服务端验证用户名密码
4. 服务端生成JWT令牌
5. 前端存储令牌，后续请求携带令牌
```

| P1 | 响应时间>1s | 短信、邮件 |

## 十二、总结

本架构设计文档定义了"城迹"项目的完整技术架构，包括：

1. **总体架构**：前后端分离，统一服务端
2. **技术架构**：UniApp + Vue 3 + Koa + MySQL + Redis + MongoDB
3. **部署架构**：容器化部署，支持水平扩展
4. **数据架构**：多数据存储，合理分工
5. **安全架构**：多层防护，全面保障
6. **性能架构**：多级缓存，异步处理
7. **扩展性架构**：支持水平扩展和微服务化

该架构设计满足项目当前需求，并为未来发展预留了扩展空间。
