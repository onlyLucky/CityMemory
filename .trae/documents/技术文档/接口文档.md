# 城迹服务端 - API接口文档

## 一、文档概述

### 1.1 文档说明
本文档定义"城迹"项目服务端API接口规范，包括小程序端和后台管理系统端的所有接口。服务端统一为小程序和后台管理系统提供API服务。

### 1.2 基础信息

| 项目 | 内容 |
|------|------|
| API基础路径 | `/api/v1` |
| 协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |

### 1.3 通用响应格式

#### 成功响应
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

#### 失败响应
```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

### 1.4 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未授权或token无效 |
| 403 | 已登录无权限 |
| 404 | 资源不存在 |
| 405 | 方法不存在 |
| 409 | 数据冲突，账号已存在 |
| 422 | 请求参数验证失败 |
| 429 | 超出请求频率 |
| 500 | 服务器内部错误 |
| 502 | 网关错误 |
| 503 | 服务不可用 |
| 504 | 网关超时 |
